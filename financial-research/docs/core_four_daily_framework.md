# 核心股票日度监控框架

对象：东鹏饮料 `605499.SH`、美的集团 `000333.SZ`、立讯精密 `002475.SZ`、福耀玻璃 `600660.SH`、联泰环保 `603797.SH`。

目标不是每天给买卖指令，而是用同一套证据框架回答三件事：

1. 今天发生了什么。
2. 为什么涨跌。
3. 是否改变原来的观察结论。
4. 明天应该按什么情景观察和应对。

## 报告时间

建议每天 `17:30 Asia/Shanghai` 生成报告。

理由：

- A股 15:00 收盘后，17:30 已能拿到完整收盘价、成交额、资金流、日K。
- 第一批盘后公告、新闻和评级信息通常已经出现。
- 过早会漏掉收盘确认，过晚不利于当天复盘。

## 每日数据层

每只股票固定抓取：

- `quote`：收盘价、涨跌幅、开高低收、成交量、成交额。
- `calc-index`：PE、PB、市值、换手率、量比、5日/10日/半年涨跌幅、资金流。
- `kline`：最近 60 个交易日前复权日K，计算 MA5、MA10、MA20、RSI14。
- `capital`：大/中/小单资金分布。
- `news`：最近新闻，按业绩、公告、资本动作、评级、行业、舆情分类。
- `filing`：最新公告，识别分红、回购、增减持、业绩、重大事项。

## 分析层

每只股票按同一顺序分析：

1. **价格行为**：涨跌幅、是否涨跌停、收盘在日内高低位置、是否放量。
2. **趋势位置**：价格相对 MA5/MA10/MA20，判断短线止跌、趋势修复或继续走弱。
3. **资金行为**：主力资金净流入/流出，大小单结构，判断是机构推动还是情绪追涨。
4. **催化来源**：新闻、公告、评级、行业、宏观、舆情，区分真实基本面变化和短期事件。
5. **原假设变化**：是否影响核心投资问题，是否需要提升/降低关注级别。

## 表现形式

默认输出 Markdown，结构如下：

- 顶部总览表：核心股票一屏比较。
- 今日异动排序：按绝对涨跌幅和成交量异常排序。
- 明日策略总表：每只股票给出策略标签、突破确认、回踩观察、失效条件。
- 单股卡片：价格、趋势、资金、催化、结论。
- 明日观察点：每只股票 1-3 条关键价位或事件。
- 明日策略：每只股票使用“突破、回踩、失效”三段式策略。
- 风险提示：数据缺失、新闻滞后、非投资建议。

## 判断标签

日度报告使用以下标签，避免直接给买卖指令：

- `趋势修复`：站上 MA10，且接近或站上 MA20。
- `强势延续`：价格在 MA5/MA10/MA20 上方，成交量不弱。
- `超跌反弹`：涨幅大但仍低于 MA20。
- `继续走弱`：跌破 MA5/MA10，且资金流出。
- `事件驱动`：涨跌主要由新闻/公告/舆情触发。
- `等待验证`：价格动了，但缺少基本面证据。

## 明日策略标签

日报必须给出明日策略，但策略不等于买卖指令。默认使用以下标签：

- `顺势观察`：趋势已修复且资金配合；重点看缩量回踩和延续性。
- `修复确认`：刚站上短期均线但未完成中期修复；重点看 MA20 或前高压力。
- `防守等待`：仍在弱势或资金流出；重点看是否止跌并收回 MA10。
- `中性观察`：方向不明确；重点看收盘位置和资金是否同步。
- `等待数据`：行情、资金或K线数据不完整，不能输出有效策略。

每个策略必须包含三条可验证条件：

1. **突破确认**：例如放量站上 MA20 或突破今日高点。
2. **回踩观察**：例如回踩 MA10/MA20 不破且缩量。
3. **失效条件**：例如跌破今日低点且资金继续流出。

## 更新纪律

- 同一数据优先使用 Longbridge CLI。
- 如果 Longbridge 新闻为空或明显滞后，再标注为外部搜索补充。
- 除权除息、送转股、停复牌必须先排除，避免误判大涨大跌。
- 每日报告不做仓位建议，不输出“必涨/必跌”。明日策略只写情景条件，不写“必须买入/卖出”。
